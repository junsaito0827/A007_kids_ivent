# ADR-0001: 年齢フィルタ導入と英語UI切替を当面実施しない

- Status: Accepted
- Date: 2026-01-02
- Owners: Jun
- Related:
  - Spec: `doc/spec/current.md`
  - Draft/Notes: `doc/base/kids_outing_flows_v0.2.md`

## 1. Context（背景 / 課題）
ユーザーページ上での利便性向上として、
- 年齢カテゴリによる絞り込み（例：「未就学児OK」「小学生以上」）
- 英語UIへの切替（在留外国人向け）

を検討した。しかし、本プロジェクトはMVP段階であり、運用負荷と誤判定リスクが信用毀損に直結する。

## 2. Decision（決定事項）
以下は当面実施しない（却下）：
- 年齢カテゴリ（未就学児/小学生以上等）によるフィルタ導入
- 英語UI切替（日本語/Englishトグル）

現行仕様（v0.2）を維持し、運用・品質の安定を優先する。

## 3. Options Considered（検討した選択肢）
### Option A: 年齢フィルタを導入（booleanタグ方式）
- 概要：`preschoolOk` / `elementaryPlusOk` 等のタグをデータに持たせ、UIで絞り込み。
- Pros：検索体験の改善、離脱低減の可能性。
- Cons：タグ付け運用が増える。未設定が多いと0件が増える。誤判定がクレームに直結。

### Option B: 年齢フィルタを導入しない（現仕様維持）
- Pros：運用負荷を増やさず、誤判定リスクを回避。MVPのスコープ管理が容易。
- Cons：未就学児不可イベントの除外を事前にできない。ユーザーが詳細確認を要する。

### Option C: 英語UI切替を導入
- 概要：UI文言を辞書化し、トグルまたは`/en`ルーティングで切替。
- Pros：英語話者の利用障壁を下げる可能性。
- Cons：UIだけ英語で本文が日本語の場合、体験が中途半端。翻訳品質/免責/責任が重くなる。
  （ブラウザ翻訳機能の存在により、費用対効果が相対的に低い）

## 4. Rationale（理由 / 判断基準）
判断基準は以下を優先した：
- 信頼性：誤情報・誤判定・誤配信は信用を即時に毀損する
- 運用負荷：タグ付け/翻訳の継続運用がMVP段階では重い
- スコープ管理：フロー（CSV→Staging→レビュー→Publish）を安定させることが先

結果として、機能追加よりも「Publishedのみ公開」「Canonical徹底」「0件回避導線」の品質を優先する。

## 5. Consequences（影響 / トレードオフ）
- 良い影響：
  - 仕様がシンプルになり、運用事故のリスクが下がる
  - Agent（Cursor/Codex）の実装暴走を抑制しやすい（ルール化可能）
- 悪い影響：
  - 年齢での事前絞り込みができず、ユーザーが手動で確認する必要がある
  - 英語話者の体験はブラウザ翻訳に依存する

## 6. Implementation Notes（実装・運用メモ）
- `doc/spec/current.md` の Out of Scope に本決定を明記する（既に明記済みの前提）。
- `.cursor/rules/10_spec_guardrails.md` に「勝手にやらない」を明記する（既に明記済みの前提）。
- 追加実装は不要。もし途中で実装した痕跡がある場合は削除して整合を取る。

## 7. Revisit Criteria（見直し条件）
以下が満たされた場合に再検討する：
- Eventsのデータ品質（年齢・対象の明確化）が一定以上（例：欠損率 < 10%）になった
- 運用体制（レビュー担当/頻度）が確立し、タグ付け・翻訳の継続が現実的になった
- 英語話者の流入が定量的に観測され、改善ROIが見込める（KPIベース）
